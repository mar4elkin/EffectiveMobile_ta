## Система разграничения прав доступа
В системе используется ролевая модель доступа.

- Каждый пользователь имеет роль access.Role.

- У каждой роли есть:

  - название,

  - код,

  - уровень доступа.

- Уровни ролей ранжируются по приоритету:

    1 - максимальный уровень доступа (администратор), далее по возрастанию, то есть чем больше число, тем ниже уровень доступа.

### Логика отображения контента

В настройках каждого элемента контента указывается роль, начиная с которой контент доступен пользователю.

Пример структуры ролей:

- admin - уровень 1

- manager - уровень 2

- viewer - уровень 3

Если в настройках контента указано, что он доступен для роли manager, тогда:

- пользователь с ролью manager видит контент,

- пользователь с ролью admin (более высокий уровень) также видит контент,

- пользователь с ролью viewer (более низкий уровень) не видит контент.
